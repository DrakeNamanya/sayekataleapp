Before starting, ensure you have:

Create Google Play Developer Account

1.1 Register for Developer Account

Visit https://play.google.com/console and sign in with your Google account.

  • Go to Google Play Console
  • Click "Sign up" or "Get started"
  • Accept Developer Distribution Agreement
  • Pay $25 one-time registration fee
  • Complete account verification (may take 24-48 hours)

1.2 Complete Developer Profile

  • Add developer name: "SayeKatale" or your organization name
  • Add contact email: admin@sayekatale.com
  • Add physical address (required)
  • Add phone number
  • Complete identity verification if prompted

Create New App in Play Console

2.1 Start App Creation

  • Click "Create app" button
  • Enter app name: SayeKatale - Demand Meets Supply
  • Select default language: English (United States)
  • Choose app type: App (not Game)
  • Choose free or paid: Free

Publishing Updates & Changes

10.1 Creating Update APK

When you need to publish an update:

Version Code Rules

Each new APK must have a higher version code than the previous one:

Release Version Name Version Code
Initial 1.0.0 1
Bug Fix 1.0.1 2
Minor Update 1.1.0 3
Major Update 2.0.0 4

10.2 Update Version Numbers

Edit your pubspec.yaml file:

version: 1.0.1+2 # Format: major.minor.patch+buildNumber # Increase +buildNumber for each release

10.3 Build Updated APK

cd /home/user/flutter_app flutter build apk --release

10.4 Upload Update

  • Go to Play Console ‚Üí Production
  • Click "Create new release"
  • Upload new APK
  • Add release notes describing changes
  • Review and submit

10.5 Update Types

Update Type When to Use Example
Patch (x.x.X) Bug fixes, minor improvements 1.0.1 ‚Üí 1.0.2
Minor (x.X.x) New features, enhancements 1.0.0 ‚Üí 1.1.0
Major (X.x.x) Major redesign, breaking changes 1.0.0 ‚Üí 2.0.0

10.6 Staged Rollout (Recommended)

For safer updates, use staged rollout:

  • Start with 20% of users
  • Monitor for crashes and ratings
  • Gradually increase to 50%, then 100%
  • Pause rollout if issues detected
  • Can halt and rollback if critical issues found

Post-Launch Management

11.2 Respond to Reviews

Review Response Best Practices

  • Respond to reviews within 24-48 hours
  • Thank users for positive feedback

11.3 Handle Crashes

When crashes are reported:

  • Review crash reports in Play Console
  • Identify affected Android versions
  • Reproduce and fix the issue
  • Test thoroughly
  • Release update quickly

11.4 Optimize Store Listing

Based on performance data:

  • Update screenshots to show new features
  • Refresh app description
  • A/B test different icons (Play Console feature)
  • Improve feature graphic
  • Add promo video (optional)

Common Errors

Solution: Increase version code in pubspec.yaml

Solution: Ensure you're building with --release flag

Solution: Don't change package name after first upload
